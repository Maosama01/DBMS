Understanding the foundation of database management systems.

1. Core Data Models

Data models define how data is connected and processed. Click on a model to explore its structure and example.

Relational Model (SQL)

Data is organized into two-dimensional tables (relations) with rows (tuples) and columns (attributes). Based on set theory.

Example: Student Table
| ID | Name | Course_ID |
| 1 | Alice | C101 |
| 2 | Bob | C102 |

Hierarchical Model

Data is organized in a tree-like structure. A parent record can have multiple child records (1:N relationship), but a child can have only one parent.

Example: Department Structure
(Root) Company
   L- Department A (Parent)
      L- Team 1 (Child)
      L- Team 2 (Child)

Network Model

An extension of the Hierarchical model, allowing many-to-many relationships (N:M). Records are linked using pointers.

Example: Students and Courses
Student A --(enrolled in)--> Course X
Student B --(enrolled in)--> Course X
Student B --(enrolled in)--> Course Y

Document Model (NoSQL)

Stores data in flexible, semi-structured document formats (like JSON or BSON). Ideal for rapid iteration and complex nested data.

Example: Product Document
{
  "product_id": 42,
  "specs": { "color": "blue", "weight": "2kg" },
  "tags": ["electronics", "gadget"]
}
